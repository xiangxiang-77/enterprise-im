# 企业即时通讯项目说明

## 项目要做什么

本项目不是单纯 Web 前端。最终目标是一套企业级即时通讯系统，当前仓库 `im-ui` 只是已有 Web 页面前端原型。

完整系统范围：

- Web 前端：按现有界面完成会话、单聊、群聊、通讯录、文件、设置、登录、搜索等页面。
- Java 服务端：提供账号、组织架构、好友、群聊、消息、文件、后台管理、审计、推送、音视频信令等能力。
- 后台管理：管理企业组织、用户好友关系、群聊、消息审计、资源文件、工作台、系统配置、角色权限。
- 桌面端：Qt 5.9.3，交付 Windows exe。
- 移动端：Flutter，交付 Android apk。
- 通信层：IM 消息走 TCP Socket 长连接；HTTP/REST 只做管理、登录、文件元数据、后台操作等非实时能力。
- 音视频：实时音视频用 PJSIP；视频中转/穿透服务用 coturn。
- 交付物：源代码、exe、apk、服务端 Docker 镜像、skills 文档库。

## 当前仓库现状

当前 `im-ui` 是 React + Vite + TypeScript + Tailwind + Radix/shadcn 风格组件项目。

已存在：

- 会话列表、聊天页、群设置、联系人、好友申请、个人资料、搜索、文件管理、设置、登录/启动页等页面。
- Zustand mock 状态、Faker mock 数据。
- 文本/图片/语音/文件/名片等消息 UI 组件。
- 图片查看/编辑、联系人选择、通知设置等局部能力。

主要缺口：

- 仍是 mock 数据，没有真实 Java API、TCP Socket、鉴权、持久化。
- 中文文案多处编码损坏，需统一修复。
- 后台管理端不存在。
- Qt 桌面端不存在。
- Flutter 移动端不存在。
- PJSIP/coturn 音视频链路不存在。
- Docker 镜像、自动化测试、CI/CD、交付打包流程不存在。

## 怎样才算做好

按 2026-05-24 前交付，最低完成标准：

1. 源代码完整
   - Web 前端、Java 服务端、后台管理、Qt 桌面端、Flutter 移动端都有独立工程。
   - 代码可构建、可运行、配置清晰。

2. IM 核心闭环可用
   - 用户登录、Token 鉴权、多端登录状态。
   - TCP Socket 长连接在线、心跳、断线重连、消息 ACK、离线消息同步。
   - 单聊、群聊、文本、图片、语音、文件、名片、系统消息可发送和接收。
   - 已读未读、撤回、编辑、转发、收藏、点赞、正在输入、红点、免打扰、置顶可用。

3. 组织和关系链可用
   - 企业组织架构、部门、员工、通讯录。
   - 好友申请、好友关系、黑名单、备注。
   - 群创建、群成员、群主转让、群公告、群设置。

4. 文件和资源可用
   - 文件上传下载、图片原图、多图发送、文件预览、断点续传/失败重试。
   - 后台可管控类型、大小、存储、过期清理。

5. 音视频可用
   - PJSIP 完成语音/视频呼叫、接听、挂断、忙线、超时。
   - coturn 完成 NAT 穿透/中转配置。
   - 服务端保存通话记录和信令状态。

6. 后台管理可用
   - 数据大盘、组织架构、好友关系、群管理、消息审计、资源管理、工作台配置、系统配置、角色权限。
   - 管理操作有日志审计。

7. 客户端交付可验证
   - Web 可构建并访问。
   - Qt 5.9.3 可打包 exe。
   - Flutter 可打包 apk。
   - 服务端可构建 Docker 镜像并启动。

8. 测试达标
   - 服务端单元/集成测试。
   - TCP 协议测试、消息可靠性测试。
   - Web/后台 E2E 或核心页面冒烟测试。
   - 桌面端/移动端核心流程手测记录。
