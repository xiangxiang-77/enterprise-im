# Skill: 关键技术路径设计

## 目标

提前解决最容易卡死项目的技术链路。

## 必须打通的路径

1. Web 登录 -> 获取 Token -> 建 TCP 连接 -> 鉴权成功。
2. A 发送文本 -> 服务端 ACK -> B 收到 -> B 已读 -> A 看到已读。
3. 群消息 -> 服务端 fanout -> 离线用户补拉。
4. 图片/文件 -> 上传对象存储 -> 消息携带 fileId -> 对端下载。
5. 撤回/编辑 -> 服务端校验权限 -> 事件广播 -> 客户端更新 UI。
6. 音视频 -> IM 信令 -> PJSIP 呼叫 -> coturn 中转 -> 通话记录。
7. 后台审计 -> 查询消息/资源/操作日志 -> 权限控制。

## TCP 协议最低字段

- `version`
- `type`
- `requestId`
- `from`
- `to`
- `conversationId`
- `timestamp`
- `payload`
- `signature` 或鉴权上下文

## 完成标准

- 每条关键路径可用脚本或 UI 演示。
- 失败场景可重试、可观测、可排查。
