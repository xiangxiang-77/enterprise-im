# Skill: 需求拆解

## 目标

把 `.docx` 和用户补充要求拆成可验收任务。

## 输入

- `功能清单.docx`
- `后台管理清单.docx`
- 用户补充：Java 服务端、TCP Socket、PJSIP、coturn、Qt 5.9.3、Flutter、Docker、2026-05-24 交付。

## 输出

- 模块清单。
- 验收标准。
- 优先级。
- 依赖关系。
- 未确认问题。

## 流程

1. 按端拆分：Web、后台、Java 服务端、Qt、Flutter、音视频、测试、交付。
2. 按业务拆分：账号、组织、好友、会话、消息、群聊、文件、资源、审计、配置。
3. 给每个需求补验收句式：`当...时，系统应...，并可通过...验证`。
4. 标注 MVP、P1、P2。
5. 更新 `codex.md` 进度和下一步。

## 完成标准

- 每个需求有目标端、接口/协议依赖、验收方式。
- 没有“做一下”“优化一下”这种不可验收项。
