# Skill: 总体设计

## 目标

确定企业 IM 全系统架构和模块边界。

## 推荐架构

- Web 前端：React + Vite。
- 后台管理：React/Vue 管理端。
- Java 服务端：Spring Boot + Netty。
- DB：MySQL/PostgreSQL。
- 缓存/在线状态：Redis。
- 文件：MinIO/S3。
- 桌面端：Qt 5.9.3。
- 移动端：Flutter。
- 音视频：PJSIP + coturn。

## 必画/必写

- 系统上下文图。
- 服务端模块图。
- TCP 消息流。
- 音视频信令流。
- 文件上传下载流。
- 管理后台权限模型。

## 关键边界

- TCP 只承载实时消息和状态事件。
- HTTP 承载登录、后台、查询、文件元数据。
- 文件二进制走对象存储，不塞进 TCP 消息体。
- PJSIP 做媒体能力，IM 服务端只管账号映射和信令。

## 完成标准

- 新人可按架构图知道每个端、每个服务做什么。
- 每个核心链路有入口、出口、失败处理。
