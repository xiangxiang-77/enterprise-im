# 关键技术路径设计 Skill

## 1. TCP Socket 协议

### 帧格式
每帧为单行 JSON，以 `\n` 分隔：
```
{"type":"AUTH","payload":{...}}\n
{"type":"TEXT","to":"...","conversationId":"...","payload":{...}}\n
```

### 帧类型 (12种)
| Type | 方向 | 用途 |
|------|------|------|
| AUTH | C→S | 登录认证 |
| AUTH_ACK | S→C | 认证响应 |
| PING | C→S | 心跳 (30s) |
| PONG | S→C | 心跳响应 |
| TEXT | C→S | 发送消息 |
| TEXT_DELIVER | S→C | 消息送达 |
| TYPING | C→S | 正在输入 |
| TYPING_DELIVER | S→C | 输入状态推送 |
| CALL_INVITE | C→S | 发起通话 |
| CALL_UPDATE | C→S | 通话状态变更 |
| CALL_EVENT | S→C | 通话事件推送 |
| ACK | 双向 | 消息已读确认 |

### 实现要点 (Java)
```java
// TcpMessageHandler.java
// 1. 读取字节流 → 按 \n 分割 → 解析 JSON
// 2. 根据 type 字段路由到对应处理器
// 3. 认证帧验证 JWT token
// 4. 非认证帧检查 session 绑定
```

### 实现要点 (Qt C++)
```cpp

```

## 2. SIP 音视频通话

### 通话建立流程
```
1. Qt/Flutter → API: GET /api/calls/media-config (获取SIP配置)
2. Qt/Flutter → TCP: CALL_INVITE (告知对方通话意图)
3. 双方 ← Asterisk: SIP INVITE (媒体协商)
4. coturn: TURN 分配中继地址
5. RTP 媒体流直连 (通过 TURN 可选)
6. 挂断: CALL_UPDATE(hangup) + SIP BYE
```

### SIP 配置项
```yaml
# application.yml
sip:
  domain: enterprise-im.local
  realm: asterisk
  registrar: sip:127.0.0.1:5060
  password: enterprise-im-sip-secret
  android-registrar: sip:127.0.0.1:5060;transport=tcp
turn:
  url: turn:10.200.38.225:3478
  username: enterprise-im
  password: enterprise-im-secret
```

## 3. 文件上传 (分块)

### 流程
```
Client → POST /api/files/chunk-session (fileSize, fileName)
       ← sessionId, chunkSize, totalChunks
Client → POST /api/files/chunks (sessionId, chunkIndex, file data)
       ← {received: N}
  ... (循环上传所有块)
Client → POST /api/files/chunk-complete (sessionId)
       ← {fileUrl, fileId}
```

### Qt 实现
```cpp
// MainWindow.cpp — startChunkUpload → uploadNextChunk → completeChunkUpload
// 默认块大小: 1MB
// 失败重试: 最多3次
// 上传队列: 离线时暂存，恢复后自动续传
```
