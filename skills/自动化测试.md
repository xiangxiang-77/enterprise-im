### 测试模式
```java
@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
@AutoConfigureMockMvc
class MyTest {
    @Autowired MockMvc mockMvc;
    @Autowired JdbcTemplate jdbcTemplate;
    @Autowired ObjectMapper objectMapper;

    // 每次测试后清理
    @AfterEach
    void cleanup() {
        jdbcTemplate.update("DELETE FROM xxx WHERE id LIKE 'test_%'");
    }

    // 认证辅助
    private String adminToken() throws Exception {
        var result = mockMvc.perform(post("/api/admin/auth/login")
                .contentType("application/json")
                .content("{\"phone\":\"18800000000\",\"password\":\"admin123\"}"))
            .andExpect(status().isOk())
            .andReturn();
        return objectMapper.readTree(result.getResponse().getContentAsString())
            .path("data").path("token").asText();
    }
}
```

### 关键测试原则
1. 使用 H2 内存数据库，测试间完全隔离
2. 不 mock 数据库——测试真实 SQL 逻辑
3. 管理端测试验证四角色权限
4. 每个测试独立，不依赖执行顺序

## 全量验证

### verify.ps1 检查项

**必选 (失败会阻止交付):**
- [x] Backend tests: `mvn test` 50 测试全通过
- [x] Web build: `npm run build` 无错误
- [x] Delivery artifacts: JAR/EXE/APK/文档 存在

**可选 (阻塞但不阻止):**
- [ ] Docker compose runtime: 健康检查
- [ ] Docker live call: 通话认证连通性
- [ ] TURN allocation: coturn 中继分配
- [ ] Qt toolchain build: VS2017 编译
- [ ] Flutter APK build: APK 构建

### 运行方式
```powershell
# 完整验证 (含 Docker)
scripts\verify.ps1

# 跳过 Docker (纯代码验证)
scripts\verify.ps1 -SkipDocker

# 跳过原生构建 (快速验证)
scripts\verify.ps1 -SkipNativeBuild
```
