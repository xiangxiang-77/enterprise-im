# 数据库设计 Skill

## 数据库概览
- **数据库**: MySQL 8.0 (生产), H2 (测试)
- **迁移工具**: Flyway (Spring Boot 集成)
- **迁移文件**: `im-server/src/main/resources/db/migration/`
- **ORM**: 无，使用 JdbcTemplate + RowMapper

## Migration 文件规范

### 命名
```
V{序号}__{snake_case描述}.sql
```
示例: `V18__department_permissions.sql`

### 内容要求
- 每个 migration 独立可重入
- 使用 `CREATE TABLE IF NOT EXISTS`
- 新字段用 `ALTER TABLE ADD COLUMN` + 默认值
- 禁止修改已有 migration（已发布后）

### 表设计模板
```sql
CREATE TABLE IF NOT EXISTS table_name (
    id          VARCHAR(64)  PRIMARY KEY,
    name        VARCHAR(255) NOT NULL,
    description TEXT,
    owner_id    VARCHAR(64)  NOT NULL,
    enabled     BOOLEAN      NOT NULL DEFAULT TRUE,
    sort_order  INT          NOT NULL DEFAULT 0,
    created_at  BIGINT       NOT NULL,
    updated_at  BIGINT       NOT NULL,
    INDEX idx_owner (owner_id),
    CONSTRAINT fk_owner FOREIGN KEY (owner_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## DAO 模式
```java
// 查询
jdbcTemplate.query("SELECT id, name FROM users WHERE id = ?",
    (rs, rowNum) -> new UserDto(rs.getString("id"), rs.getString("name")),
    userId);

// 写入
jdbcTemplate.update("INSERT INTO x(id, name) VALUES (?, ?)", id, name);

// 更新
jdbcTemplate.update("UPDATE x SET name = ? WHERE id = ?", name, id);
```

## 测试数据库
- 使用 H2 内存库: `jdbc:h2:mem:admin-test;MODE=PostgreSQL`
- 每个测试类自动执行完整 Flyway 迁移
- 测试数据通过 seed SQL 或 Java 代码插入

## 注意事项
- ID 使用 UUID 字符串 (`app_`/`dep_`/`msg_` 等前缀)
- 时间戳统一用 `BIGINT` (毫秒)
- 外键可延迟但不强制生产使用
- Migration 编号连续无跳号
