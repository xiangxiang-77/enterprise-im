# 需求拆解 Skill

## 适用场景
将用户提出的模糊需求转化为可执行的、可验证的开发任务。

## 项目上下文
企业级 IM 系统，四端架构：
- **后端**: Java 8 + Spring Boot 2.7, MySQL 8.0, Redis, MinIO, Asterisk/coturn
- **前端**: React 18 + TypeScript + Vite + shadcn/ui
- **桌面端**: Qt 5.9.3 (msvc2017_64), C++17, PJSIP/PJSUA
- **移动端**: Flutter 3.x (Dart), Android, PJSIP2 JNI

## 工作流程

### Step 1: 需求澄清
向用户确认以下关键信息，一次只问一个问题：
- 涉及哪个/哪些端？（后端/Web前端/Qt桌面/Flutter移动）
- 功能边界：做什么，不做什么
- 前置条件：需要什么配置/依赖
- 验收标准：怎么算做完

### Step 2: 拆解粒度
每个子任务控制在 2-5 分钟工作量，包含：
- 精确文件路径
- 具体改动内容（完整代码，不含"TODO"）
- 验证方式（跑什么命令，期望什么结果）

### Step 3: 输出格式
```
任务列表：
1. [文件名:行号] 具体改动 → 验证方式
2. [文件名:行号] 具体改动 → 验证方式
...
```

## 注意事项
- 跨端需求按 后端→前端→桌面→移动 顺序拆解
- API 变更先定义 DTO 和端点，再实现 Controller → Service → DB
- 数据库变更必须包含 Flyway migration 文件
- 前端变更必须同步更新 `api.ts` 类型定义
- Qt 变更注意信号/槽兼容性（Qt 5.9 语法限制）
- Flutter 变更注意包依赖声明（`pubspec.yaml`）

## 验证
- 每个任务完成后立即验证
- 后端: `mvn test`
- 前端: `npm run build`
- Qt: `scripts/package-qt-vs2017-webengine.ps1`
- Flutter: `flutter analyze`
- 全量: `scripts/verify.ps1 -SkipNativeBuild`
