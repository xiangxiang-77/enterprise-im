## 开发规范

### 分层架构
```
Controller (接收请求, 参数验证, 权限检查)
    ↓
Service (业务逻辑, 事务管理)
    ↓
JdbcTemplate (数据访问, 原生 SQL)
```

### Controller 模板
```java
@RestController
@RequestMapping("/api/admin")

        AdminSession admin = authService.requireAdmin(auth);
        authService.requireRole(admin, "SUPER_ADMIN", "OPERATOR_ADMIN");
        // ... 查询 ...
        return ApiResponse.ok(results);
    }

    @PostMapping("/xxx")
    public ApiResponse<XxxDto> create(@RequestHeader("Authorization") String auth,
                                       @RequestBody XxxRequest request) {
        AdminSession admin = authService.requireAdmin(auth);
        authService.requireRole(admin, "SUPER_ADMIN", "OPERATOR_ADMIN");
        String id = "xxx_" + UUID.randomUUID();
        jdbcTemplate.update("INSERT INTO xxx(id, name) VALUES (?, ?)", id, request.name());
        audit(admin.userId(), "XXX_CREATE", "xxx", id, request.name());
        return ApiResponse.ok(getById(id));
    }
}
```

### 权限矩阵
| 角色 | dashboard | users | org | groups | resources | audit | config | advanced |
|------|-----------|-------|-----|--------|-----------|-------|--------|----------|
| SUPER_ADMIN | RW | RW | RW | RW | RW | R | RW | RW |
| OPERATOR_ADMIN | R | RW | RW | RW | RW | - | R | R |
| SECURITY_AUDITOR | R | R | R | R | R | RW | - | - |
| READONLY_OPS | R | R | R | R | R | - | - | - |

### 配置管理 (ImProperties.java)
- 使用 `@ConfigurationProperties` 前缀绑定
- 环境变量映射: `application.yml` → Shell 变量 → `.env` 文件
- 新增 Provider 配置时同步更新 `ImProperties.java`, `application.yml`, `.env.example`

### 测试
```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@AutoConfigureMockMvc
class AdminApiTest {
    @Autowired MockMvc mockMvc;
    @Autowired JdbcTemplate jdbcTemplate;

    @Test
    void testCreateXxx() throws Exception {
        String token = adminToken();
        mockMvc.perform(post("/api/admin/xxx")
                .header("Authorization", "Bearer " + token)
                .contentType("application/json")
                .content("{\"name\":\"test\"}"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.success").value(true));
    }
}
```
